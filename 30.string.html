📌 1. What is a String?

A string is a sequence of characters. It can be a single character (e.g., 'J') or multiple characters (e.g., "JavaScript", "Tap Academy"). In JavaScript, both single characters and sequences are treated as strings.

✅ Definition: String = sequence of characters.

📌 తెలుగు : String అంటే characters యొక్క sequence. ఒకే character ('J') లేదా బహుళ characters ("JavaScript") రెండూ strings గానే పరిగణించబడతాయి.

🛠️ 2. Three Ways to Create Strings in JavaScript
// 1. Double quotes let str1 = "JavaScript"; // 2. Single quotes let str2 = 'Kalyan'; // 3. Backticks (Template literals) - allows variable interpolation let name = "Kalyan"; let greeting = `Hi, ${name}!`; // Template literal console.log(greeting); // Hi, Kalyan!
💡 Template literals (backticks) support multi-line strings and ${expression} for embedding variables.

📌 తెలుగు : Strings ని మూడు విధాలుగా create చేయవచ్చు: double quotes ("..."), single quotes ('...'), backticks (`...`). Backticks (template literals) variables ని ${variable} తో embed చేయడానికి ఉపయోగపడతాయి.

🧠 3. String Interning & String Pool (Memory)

JavaScript uses string interning: for a given string value, only one unique copy is stored in the string pool (a special area in Heap). Multiple variables can reference the same string.

let s1 = "JavaScript"; let s2 = "JavaScript"; let s3 = new String("JavaScript"); console.log(s1 === s2); // true (same reference from string pool) console.log(s1 === s3); // false (s3 is an object, not primitive string) console.log(typeof s1); // "string" console.log(typeof s3); // "object"
🔥 Benefits of Interning:
• Memory optimization (no duplicate strings)
• Faster comparison (references are compared instead of characters)
⚠️ Case Sensitivity: "Java" and "java" are different strings because JavaScript is case-sensitive.

📌 తెలుగు : JavaScript string interning వాడుతుంది: ఒక string value కి string pool (Heap లో special region) లో ఒకే unique copy ఉంటుంది. బహుళ variables ఆ string నే reference చేయగలవు. ఇది memory ని ఆదా చేస్తుంది మరియు comparison ని faster చేస్తుంది. Case sensitivity: "Java" మరియు "java" వేర్వేరు strings.

🔒 4. Strings are Immutable (Cannot Be Changed)

Immutability means once a string is created, its characters cannot be changed. You can access characters using index (like arrays), but you cannot modify them.

let str = "Hello"; console.log(str[0]); // 'H' (access works) // Attempt to change character (fails silently) str[0] = 'J'; console.log(str); // Still "Hello" (unchanged) console.log(str[0]); // Still 'H'
✅ Why Immutable? Because of string interning (shared references), changing one reference would affect others. To prevent bugs, strings are made immutable.
// To "change" a string, you must create a new one let original = "Hello"; let modified = "J" + original.slice(1); // "Jello" console.log(original); // "Hello" (unchanged) console.log(modified); // "Jello"

📌 తెలుగు : Strings immutable (మార్చలేము). ఒకసారి string create అయిన తర్వాత, దాని characters ని మార్చలేము. Access చేయవచ్చు (str[0]) కానీ modify చేయలేము. ఇది string interning వలన shared references ని protect చేయడానికి.

📦 5. Primitive Strings vs String Objects (new String())
let primitive = "Hello"; // primitive string let strObject = new String("Hello"); // String object console.log(typeof primitive); // "string" console.log(typeof strObject); // "object" console.log(primitive === strObject); // false (different types) console.log(primitive == strObject); // true (value equality)
🔥 Important: Always use primitive strings ("..." or '...'). new String() creates an object wrapper, which can cause unexpected behavior with ===.

Primitive strings are automatically converted to objects when you call methods like .toUpperCase(), then converted back.

📌 తెలుగు : Primitive strings ("...") మరియు String objects (new String()) వేర్వేరు. typeof primitive → "string", object → "object". === తో compare చేస్తే false వస్తుంది (types different). ఎప్పుడూ primitive strings ని వాడండి.

🔍 6. Accessing Characters and length Property
let str = "JavaScript"; // Access characters using bracket notation (like array) console.log(str[0]); // 'J' console.log(str[2]); // 'v' console.log(str[9]); // 't' // length property console.log(str.length); // 10 // Last character console.log(str[str.length - 1]); // 't'
💡 Strings are zero-indexed. str[0] is first character, str[str.length-1] is last character.

📌 తెలుగు : Strings zero-indexed. str[index] తో characters ని access చేయవచ్చు. str.length తో length తెలుసుకోవచ్చు. చివరి character: str[str.length-1].

📖 7. Summary: Strings Essentials
  • Definition: Sequence of characters.
  • Creation: "double", 'single', `backtick` (template literals).
  • String Interning: Unique string values stored once in string pool → memory efficient, faster comparison.
  • Immutability: Strings cannot be changed. Any operation creates a new string.
  • Case Sensitivity: "Hello" and "hello" are different.
  • Access: str[index] (read-only), str.length.
  • Primitive vs Object: Prefer primitive strings; new String() creates object wrapper.
// Quick recap let s = "JavaScript"; console.log(s.length); // 10 console.log(s[0]); // 'J' console.log(s.toUpperCase()); // "JAVASCRIPT" (new string) console.log(s); // "JavaScript" (unchanged, immutable)
