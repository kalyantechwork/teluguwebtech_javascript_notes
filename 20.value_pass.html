🔄 Pass by Value vs Pass by Reference in JavaScript

📦 Primitives (value copy) vs Objects (reference copy) | Stack & Heap Memory | Reference variables — by Kalyan (Telugu WebTech)
📌 1. Pass by Value (Primitives - Number, String, Boolean, etc.)

When you assign a primitive value (number, string, boolean, undefined, null) from one variable to another, JavaScript creates a copy of that value. This is called pass by value. The two variables are independent.

let a = 100; let b = a; // Value 100 is COPIED to b console.log(a, b); // 100 100 b = 200; // Changing b does NOT affect a console.log(a, b); // 100 200
🔥 Analogy: Like making a photocopy of a document. Changes to the photocopy do NOT affect the original. Independent copies.

Primitive types in JavaScript: Number, String, Boolean, Undefined, Null, Symbol, BigInt.

📌 తెలుగు : Primitive values (number, string, boolean, etc.) ని ఒక variable నుండి మరొక variable కి assign చేసినప్పుడు, JavaScript ఆ value యొక్క copy ని create చేస్తుంది. దీన్నే pass by value అంటారు. రెండు variables independent గా ఉంటాయి. ఒకదాన్ని మారిస్తే మరొకటి మారదు.

📦 2. Pass by Reference (Objects - including Arrays, Functions)

When you assign an object from one variable to another, JavaScript copies the reference (memory address), not the object itself. Both variables point to the same object. This is called pass by reference.

let car1 = { name: "BMW", cost: 75.6, mileage: 8.9 }; let car2 = car1; // Reference (address) is COPIED, not the object console.log(car1.name); // BMW console.log(car2.name); // BMW (same object) car2.name = "Kia"; // Changing via car2 affects the same object console.log(car1.name); // Kia (changed!) console.log(car2.name); // Kia
🔑 Key Insight: Both variables point to the same object in Heap memory. Modifying through either reference changes the single shared object.

📌 తెలుగు : Object ని ఒక variable నుండి మరొక variable కి assign చేసినప్పుడు, JavaScript reference (memory address) ని copy చేస్తుంది, object ని కాదు. రెండు variables ఒకే object ని point చేస్తాయి. దీన్నే pass by reference అంటారు. ఒక reference ద్వారా object ని మారిస్తే, మరొక reference ద్వారా కూడా ఆ మార్పు కనిపిస్తుంది.

🧠 3. Memory Visualization: Stack and Heap

Understanding memory is key to mastering pass by value vs reference.

  • Stack Memory: Stores primitive values and references (addresses) to objects. Fast, fixed-size.
  • Heap Memory: Stores objects (including arrays and functions). Dynamic, larger.
let a = 100; // 'a' in Stack stores value 100 let b = a; // 'b' in Stack stores copy of value 100 let obj1 = { x: 10 }; // Object in Heap (address 0x1000), 'obj1' in Stack stores 0x1000 let obj2 = obj1; // 'obj2' in Stack stores copy of address 0x1000 (same object!)
🎯 Multiple references to same object: One object can have many references (like multiple keys to the same locker).

📌 తెలుగు : Stack Memory: primitive values మరియు object addresses (references) ని store చేస్తుంది. Heap Memory: objects (arrays, functions) ని store చేస్తుంది. Object Heap లో ఒక్కసారి create అవుతుంది. Stack లో ఉన్న references (variables) ఆ address ని hold చేస్తాయి. ఒకే object ని ఎన్ని references అయినా point చేయవచ్చు.

🔑 4. Analogy: Locker and Multiple Keys
Pass by Value (Primitives)
Like having two separate lockers.
Each key opens its own locker.
Changing one locker doesn't affect the other.
Pass by Reference (Objects)
Like having two keys to the SAME locker.
Both keys open the same locker.
Change inside using one key → other key sees the change.
💡 Remember: For objects, the variable stores the address (key), not the object (locker) itself.

📌 తెలుగు : Pass by Value: రెండు వేర్వేరు lockers, రెండు వేర్వేరు keys. ఒక locker మారిస్తే మరొకటి మారదు.
Pass by Reference: ఒకే locker కి రెండు keys. ఒక key తో locker లో change చేస్తే, రెండో key కి కూడా అదే change కనిపిస్తుంది. Objects విషయంలో variable లో object కాకుండా address (key) store అవుతుంది.

⚠️ 5. Important: Reassigning a Reference Breaks the Link

If you reassign a variable to a new object, it now points to a different object. The other variable still points to the original object.

let objA = { value: 10 }; let objB = objA; // Both point to same object objB = { value: 20 }; // objB now points to NEW object console.log(objA.value); // 10 (unchanged) console.log(objB.value); // 20
🔥 Key distinction: Modifying a property (obj.property = newValue) affects the shared object. Reassigning the variable itself (obj = newObject) makes it point elsewhere without affecting other references.

📌 తెలుగు : Property modification: obj.name = "new" → shared object మారుతుంది.
Reassignment: obj = newObject → ఆ variable వేరే object ని point చేస్తుంది; ఇతర references ప్రభావితం కావు.

📞 6. Function Parameters: Same Rules Apply

When you pass a variable to a function:

  • Primitive: Pass by value (copy) → changes inside function don't affect outside variable.
  • Object: Pass by reference (address copy) → changes to object properties affect the original object.
// Primitive - pass by value function changePrimitive(x) { x = 200; // changes local copy only } let num = 100; changePrimitive(num); console.log(num); // 100 (unchanged) // Object - pass by reference function changeObject(obj) { obj.name = "Modified"; // changes original object } let car = { name: "BMW" }; changeObject(car); console.log(car.name); // "Modified" (changed!)
✅ Remember: For objects, the function receives a copy of the reference (address), so it can modify the original object.

📌 తెలుగు : Function కి argument పంపినప్పుడు:
• Primitive: value copy పంపబడుతుంది → function లో change చేస్తే original మారదు.
• Object: reference (address) copy పంపబడుతుంది → function లో object properties మారిస్తే original object మారుతుంది.

📊 7. Summary: Pass by Value vs Pass by Reference
✅ Pass by Value
• Works with: Primitives (Number, String, Boolean, etc.)
• Copies the actual value
• Independent copies
• Changing one doesn't affect the other
• Stored directly in Stack
🔄 Pass by Reference
• Works with: Objects (including Arrays, Functions)
• Copies the reference (address)
• Multiple references to same object
• Changing via any reference affects the shared object
• Object in Heap, reference in Stack
// Quick reference let p = 10; let q = p; // Pass by value (copy) q = 20; // p is still 10 let objP = { val: 10 }; let objQ = objP; // Pass by reference (address copy) objQ.val = 20; // objP.val also becomes 20
🧠 Kalyan's Advice: Always visualize Stack (for references/values) and Heap (for objects). This mental model will save you hours of debugging.

📌 తెలుగు సారాంశం :
• Pass by Value: Primitives (numbers, strings, booleans). Value copy. Independent. Stack లో store.
• Pass by Reference: Objects, Arrays, Functions. Reference (address) copy. Shared object. Object Heap లో, reference Stack లో.
• ఎప్పుడూ Stack మరియు Heap ని visualize చేసుకోండి — ఇది JavaScript debugging లో చాలా ఉపయోగపడుతుంది.
