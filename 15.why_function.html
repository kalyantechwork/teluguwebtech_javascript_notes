📌 1. The DRY Principle: Don't Repeat Yourself

DRY stands for Don't Repeat Yourself — one of the most fundamental principles in programming. It states: every piece of knowledge or logic should have a single, unambiguous representation within a system. In simple words: if you find yourself writing the same code multiple times, stop and think — write it once and reuse it.

🔥 Repeating code is bad (hard to maintain, error-prone). Reusing code is great (single source of truth, easy updates).

Functions are the primary way to achieve DRY in JavaScript.

📌 తెలుగు : DRY అంటే Don't Repeat Yourself — ఇది programming లో మౌలిక సూత్రాలలో ఒకటి. అర్థం: ఒకే logic ని మళ్ళీ మళ్ళీ రాయకండి. ఒకసారి రాయండి, తిరిగి తిరిగి reuse చేయండి. Code ని repeat చేయడం వలన maintain చేయడం కష్టం, bugs ఎక్కువవుతాయి. Functions ద్వారా DRY principle ని achieve చేస్తాం.

❌ 2. Without Functions: Repetitive Code (Violates DRY)

Let's calculate grade for a student based on percentage. Grading criteria:

  • Grade A: 90% and above
  • Grade B: 80% – 89%
  • Grade C: 70% – 79%
  • Grade D: 60% – 69%
  • Grade F: below 60%
// Student 1 let student1Marks = 93; let student1MaxMarks = 100; let student1Percentage = (student1Marks / student1MaxMarks) * 100; let student1Grade; if (student1Percentage >= 90) student1Grade = "A"; else if (student1Percentage >= 80) student1Grade = "B"; else if (student1Percentage >= 70) student1Grade = "C"; else if (student1Percentage >= 60) student1Grade = "D"; else student1Grade = "F"; console.log("Student 1 Grade:", student1Grade); // Student 2 - SAME LOGIC REPEATED (BAD!) let student2Marks = 62; let student2MaxMarks = 100; let student2Percentage = (student2Marks / student2MaxMarks) * 100; let student2Grade; if (student2Percentage >= 90) student2Grade = "A"; else if (student2Percentage >= 80) student2Grade = "B"; else if (student2Percentage >= 70) student2Grade = "C"; else if (student2Percentage >= 60) student2Grade = "D"; else student2Grade = "F"; console.log("Student 2 Grade:", student2Grade);
💡 For 100 students, this would be 2500+ lines of repetitive code! Violates DRY completely.

📌 తెలుగు : విద్యార్థి grade ని percentage ఆధారంగా calculate చేసే logic ని ప్రతీ student కి repeat చేస్తే ఎంత కష్టమో చూడండి. 100 students ఉంటే 2500+ lines రాయాల్సి వస్తుంది. ఇది DRY principle ని పూర్తిగా ఉల్లంఘిస్తుంది. ప్రతీసారీ ఒకే logic ని కాపీ చేయడం వలన maintain చేయడం కష్టం, bugs వచ్చే అవకాశం ఎక్కువ.

✅ 3. With Functions: Reusable, DRY-compliant Code

Extract the common logic into a function and call it for each student with different inputs.

// Reusable function to calculate and print grade function calculateGrade(marks, maxMarks) { let percentage = (marks / maxMarks) * 100; let grade; if (percentage >= 90) grade = "A"; else if (percentage >= 80) grade = "B"; else if (percentage >= 70) grade = "C"; else if (percentage >= 60) grade = "D"; else grade = "F"; console.log(`Marks: ${marks}/${maxMarks} → ${percentage.toFixed(2)}% → Grade: ${grade}`); } // Reuse the function for different students calculateGrade(93, 100); // Student 1 calculateGrade(62, 100); // Student 2 calculateGrade(32, 50); // Student 3 (64% → Grade D) calculateGrade(45, 100); // Student 4 (45% → Grade F) calculateGrade(88, 100); // Student 5 (88% → Grade B)
✅ Benefits: Write logic once, reuse infinitely. Change grading logic in one place → updates everywhere. Clean, maintainable, professional code.

Same function can be called 100 times for 100 students — no repetition!

📌 తెలుగు : ఒక function లో common logic ని ఉంచి, ప్రతీ student కి ఆ function ని call చేస్తే ఎంత సులభమో చూడండి. calculateGrade(marks, maxMarks) function ఒక్కసారి రాస్తే చాలు, ఎన్ని students అయినా reuse చేయవచ్చు. Grading logic మార్చాల్సి వస్తే ఒక్కచోట మారిస్తే సరిపోతుంది. ఇది clean, maintainable, professional code. ఇదే DRY principle యొక్క శక్తి.

🌍 4. Real-World Scenarios: Functions Everywhere

Functions are used in almost every real-world application to avoid repetition:

  • Login Validation: Validate username/password — reused across login page, session refresh, password change flows.
  • E-commerce Discount Calculation: Same discount logic applied to different cart values (5%, 10%, 50% off).
  • Tax Calculation: Calculate GST based on product price — used in checkout, invoices, order summary.
  • Form Field Validation: Check email format, phone number — reused across registration, profile edit, contact forms.
  • Date Formatting: Convert dates to user-friendly format — used everywhere in dashboards, comments, posts.
// Example: Reusable discount calculator function applyDiscount(price, discountPercent) { let discountAmount = (price * discountPercent) / 100; let finalPrice = price - discountAmount; return finalPrice; } let product1 = applyDiscount(1000, 10); // 10% off → 900 let product2 = applyDiscount(500, 5); // 5% off → 475 let product3 = applyDiscount(2000, 20); // 20% off → 1600
🧠 Kalyan's note: Whenever you find yourself copy-pasting code, stop. Think: "Can I write a function for this?" That's the mark of a good programmer.

📌 తెలుగు : ప్రతీ real-world application లో functions ని వాడతారు:
• Login validation — username/password check
• Discount calculation — different discount percentages కి same logic
• Tax calculation (GST) — checkout, invoices లో reuse
• Form validation — email, phone number format check
మీరు code ని copy-paste చేస్తున్నట్లు అనిపించినప్పుడు, ఆపండి. ఆలోచించండి: "Can I write a function for this?" అదే మంచి programmer కి గుర్తు.

📊 5. Reusability vs Repetition: Visual Comparison
// ❌ WITHOUT FUNCTION (REPETITION - BAD) // 100 students = 2500+ lines // Any change requires editing 100 places // ✅ WITH FUNCTION (REUSABLE - GOOD) function calculateGrade(marks, maxMarks) { // logic written once } // Just call for each student calculateGrade(93, 100); calculateGrade(62, 100); calculateGrade(78, 100); // ... 97 more calls (just 3 lines per student)

With functions, you also get modularity — each function does one thing, making the program easier to understand, test, and debug.

📌 తెలుగు : Repeating code (copy-paste) — 100 students కి 2500+ lines, maintenance nightmare. Functions తో — logic ఒక్కసారి రాయండి, 100 students కి కేవలం 3 lines per student. ఇది modularity ని కూడా ఇస్తుంది — ఒక్కో function ఒక్కో specific task చేస్తుంది, program ని understand, test, debug చేయడం చాలా సులభమవుతుంది.

📤 6. Return vs Print: Functions Can Also Return Values

In the example above, we printed the grade directly. But often, we want the function to return the value so the caller can decide what to do (store, print, use in another calculation).

// Function that RETURNS the grade (more flexible) function getGrade(marks, maxMarks) { let percentage = (marks / maxMarks) * 100; if (percentage >= 90) return "A"; if (percentage >= 80) return "B"; if (percentage >= 70) return "C"; if (percentage >= 60) return "D"; return "F"; } let studentGrade = getGrade(93, 100); console.log("Student 1:", studentGrade); // A // Reuse the returned value let anotherGrade = getGrade(62, 100); console.log("Student 2:", anotherGrade); // D
💡 Returning values makes functions more reusable. The caller can decide to print, store, or use the result in further calculations.

📌 తెలుగు : కొన్నిసార్లు function ని print చేయకుండా return చేయడం వలన more flexibility వస్తుంది. Caller ఆ value ని print చేసుకోవచ్చు, store చేసుకోవచ్చు, లేదా మరొక calculation లో use చేసుకోవచ్చు. return వలన function pure గా ఉంటుంది — same input ఇస్తే same output వస్తుంది.

📖 7. Summary: Functions are the solution to DRY
  • DRY Principle: Don't Repeat Yourself — write code once, reuse everywhere.
  • Violation: Copy-pasting same logic multiple times (hard to maintain, error-prone).
  • Solution: Encapsulate repetitive logic inside a function.
  • Benefits: Reusability, maintainability, single source of truth, easier debugging.
  • Function components: name, parameters (input), body (logic), return (output).
// Golden rule for beginners: // If you write the same code twice, it's time to write a function.
