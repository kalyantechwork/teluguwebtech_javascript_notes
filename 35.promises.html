📌 1. Promise Basics: Object, State, Executor Function

A Promise is an object representing the eventual completion (or failure) of an asynchronous operation. It has three states: pending, fulfilled, rejected.

const pr = new Promise((resolve, reject) => { // Executor function runs automatically when promise is created console.log("Inside executor function"); // Async operation here });
🔥 Key point: The executor function executes immediately when the promise is created (synchronously), not later.

📌 తెలుగు : Promise అనేది asynchronous operation యొక్క eventual completion ని represent చేసే object. దీనికి మూడు states: pending, fulfilled, rejected. Executor function promise create అయిన వెంటనే immediate గా run అవుతుంది.

🔄 2. resolve(), reject() and then(), catch()
let promise = new Promise((resolve, reject) => { let success = true; if (success) { resolve("Data loaded successfully"); } else { reject("Error occurred"); } }); promise .then((result) => console.log("Success:", result)) .catch((error) => console.log("Error:", error));
✅ resolve(value) → state becomes fulfilled → triggers .then() callback.
✅ reject(reason) → state becomes rejected → triggers .catch() callback.

📌 తెలుగు : resolve(value) → fulfilled state → .then() callback execute అవుతుంది. reject(reason) → rejected state → .catch() callback execute అవుతుంది.

⏳ 3. Async Operation inside Promise (setTimeout)
const boilingWater = new Promise((resolve, reject) => { console.log("Water started boiling"); setTimeout(() => { console.log("Water has been boiled"); resolve("Promise fulfilled"); }, 3000); }); boilingWater .then(() => console.log("Promise fulfilled!")) .catch(() => console.log("Promise failed"));
🔥 The setTimeout callback is registered in Web API. When timer completes, callback goes to Callback Queue (Macrotask). Then resolve puts the .then callback into Microtask Queue.

📌 తెలుగు : setTimeout callback Web API లో register అవుతుంది. Timer complete అయిన తర్వాత callback Macrotask Queue లోకి వెళ్తుంది. resolve call అయిన తర్వాత .then() callback Microtask Queue లోకి వెళ్తుంది.

⚡ 4. Microtask Queue vs Macrotask Queue (Priority)
setTimeout(() => console.log("Macrotask (setTimeout)"), 0); Promise.resolve().then(() => console.log("Microtask (Promise)")); console.log("Synchronous code"); // Output order: // Synchronous code // Microtask (Promise) // Macrotask (setTimeout)
🧠 Event Loop Priority: After each macrotask, the event loop empties the entire microtask queue before taking the next macrotask. Microtasks have higher priority.
📦 Microtask Queue: Promise.then, catch, finally, queueMicrotask
📦 Macrotask Queue: setTimeout, setInterval, I/O, UI rendering

📌 తెలుగు : Event Loop priority: ప్రతీ macrotask తర్వాత, event loop microtask queue ని పూర్తిగా empty చేస్తుంది, ఆ తర్వాతే next macrotask తీసుకుంటుంది. Microtasks కు higher priority.

🔗 5. Promise Chaining (E-commerce Workflow)
function stepOne() { return new Promise((resolve, reject) => { console.log("Step 1 started"); setTimeout(() => { console.log("Step 1 completed"); resolve(); }, 4000); }); } function stepTwo() { return new Promise((resolve, reject) => { console.log("Step 2 started"); setTimeout(() => { console.log("Step 2 completed"); resolve(); }, 3000); }); } function stepThree() { return new Promise((resolve, reject) => { console.log("Step 3 started"); setTimeout(() => { console.log("Step 3 completed"); resolve(); }, 2000); }); } function stepFour() { return new Promise((resolve, reject) => { console.log("Step 4 started"); setTimeout(() => { console.log("Step 4 completed"); resolve(); }, 1000); }); } // Promise Chaining stepOne() .then(() => stepTwo()) .then(() => stepThree()) .then(() => stepFour()) .then(() => console.log("All steps successful!")) .catch(() => console.log("Promise failed at some step"));
✅ Promise chaining eliminates callback hell. Each step runs only after the previous step's promise resolves. Error propagates to single .catch().

📌 తెలుగు : Promise chaining callback hell ని solve చేస్తుంది. ప్రతీ step మునుపటి step resolve అయిన తర్వాతే execute అవుతుంది. Error ఒకే .catch() కి propagate అవుతుంది.

⚠️ 6. Error Propagation: One .catch() for All
stepOne() .then(() => stepTwo()) .then(() => { throw new Error("Something went wrong in step 3"); }) .then(() => stepFour()) .catch((error) => console.log("Caught error:", error.message)); // The chain stops at the rejected promise and jumps to .catch()
🔥 Any reject() or thrown error inside a .then() will skip all subsequent .then() and go directly to the nearest .catch().

📌 తెలుగు : ఏ .then() లోనైనా reject() లేదా error వస్తే, తర్వాతి .then() లన్నీ skip అయ్యి, నేరుగా .catch() కి వెళ్తుంది.

📖 7. Summary: Promises & Event Loop
  • Promise = object with state (pending/fulfilled/rejected).
  • Executor function runs immediately; calls resolve (success) or reject (failure).
  • .then() handles resolved value; .catch() handles rejection.
  • Promise Chaining → sequential async operations without nesting.
  • Microtask Queue (Promises) has higher priority than Macrotask Queue (setTimeout).
  • Event Loop: after each macrotask, it empties the microtask queue before next macrotask.
// Mental model // 1. Create promise → executor runs → async operation (Web API) // 2. On completion, resolve/reject // 3. .then() callback → Microtask Queue // 4. Event Loop prioritizes Microtask Queue
