📌 1. Functions Can Return Functions (First-Class Citizens)

Because functions are objects (first-class citizens) in JavaScript, a function can return another function as its output.

function outer() { function inner() { console.log("Inside inner function"); } return inner; // returning the function itself (not calling it) } let result = outer(); // result now holds the inner function result(); // calling inner function → "Inside inner function"
🔥 Memory: The inner function object is created in Heap. Its reference is returned and stored in result. The outer function's execution context may finish, but the inner function object persists if referenced.

📌 తెలుగు : JavaScript లో functions first-class citizens (objects) కాబట్టి, ఒక function మరొక function ని return చేయగలదు. ఉదా: function outer() { return function inner() {...} }. Inner function Heap లో object గా create అవుతుంది. దాని reference return అయి variable లో store అవుతుంది. Outer function execution అయిపోయినా, inner function object persist అవుతుంది.

🔐 2. What is a Closure?

A closure is created when an inner function has access to variables of its outer function, even after the outer function has finished execution. The inner function "closes over" the outer function's scope (lexical scope).

function outer() { let outerVariable = 10; // variable in outer function's scope function inner() { console.log("Outer variable value:", outerVariable); } return inner; } let myFunction = outer(); // outer() finishes execution myFunction(); // Still prints "Outer variable value: 10"
🌟 Closure = Inner function + access to outer function's variables (lexical environment)
💡 Even though outer() has returned and its execution context is gone, the lexical scope (outerVariable) is preserved because the inner function forms a closure around it.

📌 తెలుగు : Closure అంటే: inner function, outer function యొక్క variables ని access చేయగలగడం, outer function execution అయిపోయిన తర్వాత కూడా. Inner function outer function యొక్క lexical scope మీద "close" అవుతుంది. Outer function execution context పోయినా, lexical scope (outerVariable) preserve అవుతుంది.

📦 3. Lexical Scope: Why Variables Survive

Lexical scope means that a function's scope is determined by where it is defined (not where it is called). When an inner function is returned, it brings its lexical environment (including outer variables) with it.

function createCounter() { let count = 0; // lexical variable return function() { count++; // inner function closes over 'count' console.log("Count:", count); }; } let counter = createCounter(); counter(); // Count: 1 counter(); // Count: 2 counter(); // Count: 3
🧠 Closure in action: Each call to counter() increments the same count variable, because the closure retains that variable across calls.

📌 తెలుగు : Lexical scope: ఒక function యొక్క scope అది ఎక్కడ define చేయబడిందో దానిపై ఆధారపడి ఉంటుంది (ఎక్కడ call అయిందో కాదు). Inner function return అయినప్పుడు, అది దాని lexical environment (outer variables) ని కూడా తీసుకువెళుతుంది. అందుకే counter function ప్రతీసారీ count ని increment చేయగలదు.

🔢 4. Multiple Independent Closures (Counter Example)

Each call to the outer function creates a new, independent closure with its own lexical environment.

function createCounter() { let count = 0; return function() { count++; console.log("Count:", count); }; } let counterA = createCounter(); let counterB = createCounter(); counterA(); // Count: 1 counterA(); // Count: 2 counterB(); // Count: 1 (independent!) counterB(); // Count: 2
🔥 Key insight: counterA and counterB are two separate closures. Each has its own private count variable. They don't interfere with each other.

📌 తెలుగు : Outer function ని ప్రతీసారి call చేసినప్పుడు, ఒక కొత్త, independent closure create అవుతుంది. counterA మరియు counterB వేర్వేరు closures. ప్రతీదానికి దాని స్వంత count variable ఉంటుంది. అవి ఒకదానికొకటి అడ్డుపడవు.

🛠️ 5. Practical Use Cases of Closures
  • Data Encapsulation / Private Variables: Hide variables from global scope (like count in counter example).
  • Event Handlers & Callbacks: Retain state across asynchronous operations.
  • Memoization / Caching: Store computed results to optimize performance.
  • Function Factories: Create customized functions (e.g., multipliers, loggers).
  • Module Pattern: Create modules with public and private members.
// Example: Multiplier factory using closure function multiplyBy(factor) { return function(number) { return number * factor; }; } let double = multiplyBy(2); let triple = multiplyBy(3); console.log(double(5)); // 10 console.log(triple(5)); // 15
🧠 Closure is not just a theoretical concept — it's used everywhere in JavaScript! Event listeners, timers, promises, and React hooks all rely on closures.

📌 తెలుగు : Closures యొక్క practical use cases: Data encapsulation (private variables), Event handlers & callbacks, Memoization (caching), Function factories, Module pattern. ఇవన్నీ JavaScript లో చాలా ముఖ్యమైనవి. Event listeners, timers, promises, React hooks లలో closures వాడతారు.

⚠️ 6. Closure is Created ONLY When the Inner Function is Returned/Exported

If an inner function does NOT outlive its outer function (i.e., it is called and destroyed inside), no persistent closure is formed.

function outer() { let data = "Secret"; function inner() { console.log(data); } inner(); // inner called and destroyed here — no closure } outer(); // Works, but no persistent closure because inner is not returned
✅ Closure is formed when: The inner function is returned or passed outside the outer function, thereby outliving the outer function's execution.

In the counter example, the returned function persists, creating a closure over count.

📌 తెలుగు : Closure ఏర్పడటానికి, inner function తప్పనిసరిగా return చేయబడాలి లేదా outer function బయటికి పంపబడాలి. అప్పుడే అది outer function ని "outlive" చేస్తుంది. లేకపోతే closure ఏర్పడదు.

📖 7. Summary: Closures at a Glance
  • Closure = Function + its lexical environment (outer variables).
  • Created when an inner function is returned or passed outside its outer function.
  • The inner function retains access to outer function's variables even after the outer function finishes.
  • Each call to the outer function creates a new independent closure with its own lexical scope.
  • Used for data hiding, event handlers, memoization, function factories, and more.
// Classic closure example function createGreeter(greeting) { return function(name) { console.log(greeting + ", " + name); }; } let sayHello = createGreeter("Hello"); let sayNamaste = createGreeter("Namaste"); sayHello("Kalyan"); // Hello, Kalyan sayNamaste("Kalyan"); // Namaste, Kalyan
