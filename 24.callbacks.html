📌 1. What is a Callback Function?

A callback function is a function that is passed as an argument to another function. The receiving function (called Higher-Order Function) then calls back (invokes) the callback at some point during its execution.

🔹 Higher-Order Function (host) ← receives callback as argument
🔹 Then calls the callback when needed
🔹 Callback is "called back" later
function higherOrder(callback) { // do some work... callback(); // calling the callback } function myCallback() { console.log("I am a callback!"); } higherOrder(myCallback); // passing function as argument
✅ Key Insight: In JavaScript, functions are objects (first-class citizens), so they can be passed as arguments like any other value.

📌 తెలుగు : Callback function అంటే మరొక function కి argument గా pass చేయబడే function. ఆ function ని స్వీకరించే function ని Higher-Order Function అంటారు. అది ఆ callback ని తర్వాత "call back" చేస్తుంది. Functions first-class citizens (objects) కాబట్టి, వాటిని arguments గా pass చేయవచ్చు.

🧮 2. Practical Example: Calculator with Callbacks

Instead of creating separate functions for add, subtract, multiply and calling them individually, we can create a calculator higher-order function that accepts the operation as a callback.

function add(a, b) { return a + b; } function sub(a, b) { return a - b; } function mul(a, b) { return a * b; } // Higher-Order Function function calculator(a, b, callback) { return callback(a, b); } console.log(calculator(10, 20, add)); // 30 console.log(calculator(10, 20, sub)); // -10 console.log(calculator(10, 20, mul)); // 200
🔥 Benefits: Reusability, cleaner code, and the calculator function doesn't need to know the details of each operation — it just calls the callback.

📌 తెలుగు : Calculator example: calculator(10,20,add) → add function callback గా pass అవుతుంది. calculator function ఆ callback ని call చేస్తుంది. ఇలా చేయడం వలన code reusable, clean, మరియు calculator function ప్రతీ operation యొక్క details తెలుసుకోనవసరం లేదు.

🧠 3. How Callbacks Work in Memory (Pass by Reference)

When we pass a function as a callback, we are passing its reference (memory address), not a copy. The higher-order function's parameter then points to the same function object.

function add(a, b) { return a + b; } function calculator(a, b, callback) { // callback parameter now references the SAME function as 'add' return callback(a, b); } calculator(5, 3, add); // add's reference passed to callback
💡 This works because functions are objects stored in Heap. The variable names (add, callback) are references (Stack) pointing to the same Heap object.

📌 తెలుగు : Callback ని pass చేసినప్పుడు, దాని reference (memory address) ని pass చేస్తాం (copy కాదు). Higher-order function లోని parameter అదే function object ని point చేస్తుంది. ఇది pass by reference కు ఉదాహరణ. Functions Heap లో objects, variables Stack లో references ని store చేస్తాయి.

📐 4. Real-world Example: Area Calculator
// Callback functions for different shapes function circleArea(radius) { return Math.PI * radius * radius; } function squareArea(side) { return side * side; } function equilateralTriangleArea(side) { return (Math.sqrt(3) / 4) * side * side; } // Higher-Order Function function calculateArea(value, callback) { console.log(callback(value)); } calculateArea(5, circleArea); // Circle area with radius 5 calculateArea(5, squareArea); // Square area with side 5 calculateArea(5, equilateralTriangleArea); // Triangle area with side 5
✅ The calculateArea function doesn't know about circles, squares, or triangles. It just receives a callback and delegates the area calculation to it. This is the power of callbacks.

📌 తెలుగు : calculateArea function కి circle, square, triangle గురించి ఏమీ తెలియదు. అది ఒక callback ని తీసుకొని, ఆ callback కి area calculation ని delegate చేస్తుంది. ఇది callbacks యొక్క శక్తి.

📋 5. Where are Callbacks Used in Real JavaScript?
  • Array Methods: map(), filter(), reduce(), forEach() — all accept callbacks
  • Event Handling: addEventListener('click', callback)
  • Asynchronous Operations: setTimeout, setInterval, fetch, file reading (Node.js)
  • HTTP Requests: AJAX, API calls (before Promises/async-await)
  • React / Frontend Frameworks: Event handlers, effect hooks
// Array callback example let numbers = [1, 2, 3, 4, 5]; let doubled = numbers.map(function(num) { return num * 2; }); console.log(doubled); // [2, 4, 6, 8, 10] // Event listener callback document.getElementById("btn").addEventListener("click", function() { alert("Button clicked!"); });

📌 తెలుగు : Callbacks ని ఎక్కడ ఉపయోగిస్తాం? Array methods (map, filter, reduce), Event handling (addEventListener), Asynchronous operations (setTimeout, fetch), HTTP requests, React event handlers. ఇవి JavaScript లో చాలా సాధారణం.

⏱️ 6. Synchronous vs Asynchronous Callbacks

Synchronous Callback: Executed immediately inside the higher-order function (e.g., calculator, array methods).

Asynchronous Callback: Executed later, after some operation completes (e.g., setTimeout, event listeners, API calls).

// Synchronous callback (executes immediately) function greet(name, callback) { callback(name); } greet("Kalyan", function(n) { console.log("Hello " + n); }); // Asynchronous callback (executes after delay) setTimeout(function() { console.log("This runs after 2 seconds"); }, 2000); console.log("This runs immediately"); // Output: "This runs immediately", then after 2 seconds: "This runs after 2 seconds"
🔥 Important: Callbacks are the foundation of asynchronous programming in JavaScript, allowing non-blocking operations.

📌 తెలుగు : Synchronous Callbacks: వెంటనే execute అవుతాయి (calculator, array methods). Asynchronous Callbacks: తర్వాత execute అవుతాయి (setTimeout, event listeners, API calls). Asynchronous callbacks JavaScript ని non-blocking గా చేస్తాయి.

📖 7. Summary: Callbacks in a Nutshell
  • Callback: A function passed as an argument to another function.
  • Higher-Order Function: The function that receives and calls the callback.
  • Why Callbacks? Code reusability, abstraction, and handling asynchronous operations.
  • Mechanism: Functions are objects → references are passed (pass by reference).
  • Next Steps: Promises and async/await evolved from callbacks to solve "callback hell".
// Quick recap function higherOrder(callback) { callback(); } function simpleCallback() { console.log("Callback executed!"); } higherOrder(simpleCallback); // Output: Callback executed!
