⚡ Async/Await in JavaScript | Cleaner Promises

📦 async keyword | await keyword | try/catch error handling | Promise chaining simplified — by Kalyan (Telugu WebTech)
📌 1. Evolution: Callback Hell → Promises → Async/Await

JavaScript started with callbacks (led to Callback Hell). Then Promises improved readability with .then() and .catch(). Async/Await is syntactic sugar over Promises — it makes asynchronous code look and behave more like synchronous code.

// Promises (readable but still many .then()) fetchData() .then(data => process(data)) .then(result => display(result)) .catch(error => handleError(error)); // Async/Await (even cleaner!) async function run() { try { let data = await fetchData(); let result = await process(data); display(result); } catch (error) { handleError(error); } }
✅ Async/Await does NOT replace Promises — it builds on top of them. Every async function returns a Promise.

📌 తెలుగు : JavaScript: Callback Hell → Promises → Async/Await. Async/Await అనేది Promises పై syntactic sugar. ఇది asynchronous code ని synchronous code లాగా చదవడానికి వీలు కల్పిస్తుంది. ప్రతీ async function ఒక Promise ని return చేస్తుంది.

📝 2. Basic Syntax: async function & await
// Function returning a Promise (using setTimeout) function delayedTask() { return new Promise((resolve) => { setTimeout(() => { console.log("Task completed"); resolve("Success data"); }, 2000); }); } // Async function using await async function run() { console.log("Start"); let result = await delayedTask(); // waits for promise to resolve console.log("Result:", result); console.log("End"); } run(); // Output: // Start // (after 2 sec) Task completed // Result: Success data // End
🔥 await pauses the execution of the async function until the promise resolves/rejects. It does NOT block the main thread — other code can run.

📌 తెలుగు : await async function యొక్క execution ని pause చేస్తుంది, promise resolve/reject అయ్యే వరకు. ఇది main thread ని block చేయదు — ఇతర code run అవుతుంది.

🔄 3. Converting .then() Chain to Async/Await
// Promise chain version function stepOne() { return new Promise(resolve => setTimeout(() => resolve("Step 1 done"), 1000)); } function stepTwo() { return new Promise(resolve => setTimeout(() => resolve("Step 2 done"), 1000)); } function stepThree() { return new Promise(resolve => setTimeout(() => resolve("Step 3 done"), 1000)); } // .then() chain stepOne() .then(result => { console.log(result); return stepTwo(); }) .then(result => { console.log(result); return stepThree(); }) .then(result => console.log(result)) .catch(err => console.log("Error:", err)); // ✅ Async/Await version (much cleaner!) async function executeSteps() { try { let res1 = await stepOne(); console.log(res1); let res2 = await stepTwo(); console.log(res2); let res3 = await stepThree(); console.log(res3); } catch (error) { console.log("Error:", error); } } executeSteps();
🧠 Async/Await removes the need for multiple .then() callbacks — code becomes linear and easier to read.

📌 తెలుగు : Async/Await వలన multiple .then() callbacks అవసరం లేదు. Code linear గా, చదవడానికి సులభంగా మారుతుంది.

⚠️ 4. Error Handling with try/catch
function mightFail(shouldSucceed) { return new Promise((resolve, reject) => { setTimeout(() => { if (shouldSucceed) { resolve("Success!"); } else { reject("Failure occurred"); } }, 1000); }); } async function handleAsync() { try { let result = await mightFail(true); // change to false to test error console.log("Result:", result); } catch (error) { console.log("Caught error:", error); } } handleAsync();
✅ try/catch works exactly like synchronous code. Any rejected promise or thrown error inside the try block goes to catch.

📌 తెలుగు : try/catch synchronous code లాగానే పనిచేస్తుంది. try block లో rejected promise లేదా thrown error వెంటనే catch కి వెళ్తుంది.

🛒 5. Real Example: E-commerce Workflow (Async/Await)
function addToCart() { return new Promise(resolve => setTimeout(() => resolve("Added to cart"), 1000)); } function proceedToCheckout() { return new Promise(resolve => setTimeout(() => resolve("Checkout done"), 1000)); } function makePayment() { return new Promise((resolve, reject) => { setTimeout(() => { let success = true; success ? resolve("Payment successful") : reject("Payment failed"); }, 1000); }); } async function orderFlow() { try { console.log("Starting order..."); let cart = await addToCart(); console.log(cart); let checkout = await proceedToCheckout(); console.log(checkout); let payment = await makePayment(); console.log(payment); console.log("Order completed!"); } catch (error) { console.log("Error in order:", error); } } orderFlow();
💡 This reads like synchronous code but runs asynchronously. Much easier to understand than promise chaining with multiple .then().

📌 తెలుగు : E-commerce workflow: addToCart → checkout → payment. Async/Await వలన code synchronous లాగా చదవడానికి వీలవుతుంది, కానీ అసలు execution asynchronous గా జరుగుతుంది.

📦 6. Top-level await (Modern JavaScript)

In modern JavaScript (ES2022+), you can use await at the top level of modules (not inside a function), without wrapping in an async function.

// In a module (type="module"), this works: let data = await fetchData(); console.log(data);
🔥 However, in most environments, it's still recommended to wrap your code in an async function for compatibility and clarity.

📌 తెలుగు : Modern JavaScript (ES2022+) లో top-level await (module లో) నేరుగా వాడవచ్చు, async function లో wrap చేయనవసరం లేదు. అయినా compatibility కోసం async function వాడటం మంచిది.

📖 7. Summary: Async/Await Essentials
  • async keyword before a function → function always returns a Promise.
  • await keyword pauses the async function until a Promise settles.
  • Use try/catch to handle errors (rejected promises).
  • Async/Await makes asynchronous code look synchronous → easier to read and maintain.
  • It is built on Promises — understanding Promises is still essential.
// Golden rule for async/await async function myAsyncFunction() { try { let result = await somePromise(); console.log(result); } catch (err) { console.error("Error:", err); } }
🧠 Kalyan's Advice: Master Promises first, then use async/await for cleaner code. Practice converting .then() chains to async/await with try/catch.

📌 తెలుగు సారాంశం :
• async → function ఎప్పుడూ Promise ని return చేస్తుంది.
• await → async function ని pause చేస్తుంది, promise settle అయ్యే వరకు.
• try/catch → error handling కోసం.
• Async/Await → asynchronous code ని synchronous లాగా చదవడానికి.
• ఇది Promises పై నిర్మించబడింది — Promises తెలుసుకోవడం ఇప్పటికీ ముఖ్యం.
