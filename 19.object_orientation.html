📌 1. What is "Orientation"? (Perspective / Way of Looking)

Orientation means way of looking or perspective (दृष्टिकोण). The same thing can be seen differently by different people.

🥛 A glass half-filled with water:
• Chemist → "half liquid, half gas"
• Optimist → "half full glass"
• Pessimist → "half empty glass"
• Realist → "just a glass of water"
💡 Orientation = your perspective. Object Orientation = looking at everything in the world as a collection of objects.

📌 తెలుగు : Orientation అంటే way of looking లేదా perspective (దృక్పథం). ఒకే glass of water ని chemist, optimist, pessimist, realist వేర్వేరుగా చూస్తారు. Object Orientation అంటే ప్రపంచంలోని ప్రతిదాన్ని objects యొక్క collection గా చూడటం.

📐 2. Three Fundamental Rules of Object Orientation

Rule 1: The world (and your program) is a collection of objects. (Glass, car, mobile, student, virus — everything is an object)

Rule 2: Every object belongs to a type/category. In programming, this type is called a Class. Class is imaginary (idea), objects are real.

Rule 3: Every object has two parts:

  • State / Properties (HAS) — what the object has (name, color, price, mileage)
  • Behavior / Methods (DOES) — what the object can do (start, stop, accelerate, sing, dance)
// Car object example // HAS (Properties): name, cost, mileage // DOES (Methods): start(), stop(), accelerate()
🔥 Object Orientation = Perspective of looking at world as objects + every object has properties (has) and behaviors (does).

📌 తెలుగు : Object Orientation యొక్క 3 నియమాలు:
1. ప్రపంచం (మరియు మా program) objects యొక్క collection.
2. ప్రతీ object ఒక type/category కి చెందుతుంది. Programming లో దీన్నే Class అంటారు. Class imaginary, objects real.
3. ప్రతీ object కి రెండు భాగాలు: State/Properties (HAS) మరియు Behavior/Methods (DOES).

🛠️ 3. Creating Objects in JavaScript: Object Literal Notation

There are multiple ways to create objects in JavaScript. The most common is Object Literal Notation using { }.

let car = { // Properties (HAS part) name: "BMW", cost: 75.6, mileage: 8.6, // Methods / Behaviors (DOES part) start: function() { console.log("Car is starting"); }, stop: function() { console.log("Car is stopping"); }, accelerate: function() { console.log("Car is accelerating"); } };
✅ Syntax: { key: value, key2: value2, ... }. Keys can be property names or method names. Values can be strings, numbers, booleans, arrays, or functions.

📌 తెలుగు : JavaScript లో objects ని create చేయడానికి అనేక మార్గాలు ఉన్నాయి. అత్యంత సాధారణమైనది Object Literal Notation ({ }). ఇందులో key: value pairs ఉంటాయి. Properties (HAS) మరియు Methods (DOES) రెండూ key: value గా రాస్తాం.

🧠 4. Memory View: Stack and Heap for Objects

When a JavaScript program runs, two memory segments are used:

  • Stack — stores primitive values and references (addresses)
  • Heap — stores objects (complex structures)
// When we create an object let car = { name: "BMW", cost: 75.6 }; // Memory: // 1. Object is created in HEAP with an address (e.g., 0x1000) // 2. The variable 'car' in STACK stores this address (reference)
🔥 Important: The variable holds a reference (memory address) to the object, not the object itself. Multiple variables can reference the same object.

This reference-based model is why objects are called reference types.

📌 తెలుగు : JavaScript program execute అయినప్పుడు రెండు memory segments ఉంటాయి: Stack (primitives, references) మరియు Heap (objects). Object Heap లో create అవుతుంది, దాని address (reference) Stack లో variable లో store అవుతుంది. అందుకే objects ని reference types అంటారు.

🔍 5. Accessing Object Properties & Methods (Dot Notation)

Use the dot operator (.) to access properties and methods of an object.

// Access properties console.log(car.name); // BMW console.log(car.cost); // 75.6 // Call methods car.start(); // "Car is starting" car.stop(); // "Car is stopping" car.accelerate(); // "Car is accelerating"
✅ Syntax: objectReference.propertyName or objectReference.methodName()

When a method is called, a new execution context is created on the stack, the method runs, then it's popped off.

📌 తెలుగు : Object properties మరియు methods ని access చేయడానికి dot operator (.) వాడతాం. ఉదా: car.name (property), car.start() (method call). Method call అయినప్పుడు stack లో new execution context create అవుతుంది.

📝 6. Practice: Student Object

Create a student object with the following:

  • Methods (DOES): code(), sing(), dance()
let student = { name: "Rahul", grade: "A", age: 20, code: function() { console.log("Student is coding"); }, sing: function() { console.log("Student is singing"); }, dance: function() { console.log("Student is dancing"); } }; console.log(student.name); // Rahul student.code(); // Student is coding

📚 7. Different Ways to Create Objects in JavaScript
  • Object Literal: { key: value } (most common, used in JSON)
  • new Object() Constructor: let obj = new Object();
  • Constructor Function: function Car(name) { this.name = name; }
  • ES6 Classes: class Car { constructor(name) { this.name = name; } }
💡 JSON (JavaScript Object Notation) is based on object literal syntax and is the most popular data exchange format on the internet.

We will explore classes and constructor functions in depth in upcoming classes.

📌 తెలుగు : JavaScript లో objects ని create చేయడానికి అనేక మార్గాలు: Object Literal, new Object(), Constructor Function, ES6 Classes. JSON (JavaScript Object Notation) అనేది object literal syntax పై ఆధారపడి ఉంటుంది మరియు internet లో అత్యంత ప్రజాదరణ పొందిన data exchange format.

📖 8. Summary
  • Object Orientation = perspective of looking at everything as a collection of objects.
  • Every object has properties (HAS) and methods (DOES).
  • In JavaScript, objects are created using object literals { } with key-value pairs.
  • Objects are stored in Heap memory; variables store references (addresses) in Stack.
  • Use dot notation (.) to access properties and methods.
  • Functions inside objects are called methods.
