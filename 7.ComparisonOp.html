⚖️ Comparison Operators in JavaScript

🔍 Equality (==) vs Strict Equality (===) | Type Coercion | Special cases (null, undefined, NaN) — by Kalyan (Telugu WebTech)
📌 1. Why Comparison Operators?

Programming is all about building logic based on conditions. Often we need to compare two values: check if age equals 18, if username matches, if password is correct, etc. The result of a comparison is always a Boolean (true or false).

let age = 18; console.log(age == 18); // true let actualUser = "tapacademy"; let enteredUser = "tap123"; console.log(actualUser == enteredUser); // false

JavaScript provides two main equality operators: == (equality) and === (strict equality). Understanding the difference is crucial.

📌 తెలుగు : ప్రోగ్రామింగ్ లో తరచుగా మనం రెండు విలువలను పోల్చాలి: వయస్సు 18 కి సమానమా?, username సరిపోలిందా? Comparison ఫలితం ఎప్పుడూ Boolean (true/false) లో వస్తుంది. JavaScript లో రెండు equality operators ఉన్నాయి: == (equality) మరియు === (strict equality).

📊 2. Basic Comparisons: Same Types

When both operands are of the same type, both == and === work similarly — they compare values directly.

// number with number console.log(18 == 18); // true console.log(18 === 18); // true // string with string console.log("hello" == "hello"); // true console.log("hello" === "hello"); // true // boolean with boolean console.log(true == true); // true console.log(true === true); // true

The real difference appears when comparing values of different types.

📌 తెలుగు : రెండు operands ఒకే type అయినప్పుడు == మరియు === రెండూ ఒకేలా పనిచేస్తాయి — values ని నేరుగా compare చేస్తాయి. వ్యత్యాసం different types ఉన్నప్పుడు తెలుస్తుంది.

🔄 3. Loose Equality (==) & Type Coercion (Implicit Conversion)

The == operator performs type coercion — it automatically converts values to a common type before comparing. This is also called implicit type casting.

💡 Type Coercion: When comparing different types, JavaScript converts one or both values to a common type (usually Number) and then compares.
// number vs string → string converted to number console.log(5 == "5"); // true (because "5" becomes 5) // boolean vs number → boolean converted to number (true→1, false→0) console.log(true == 1); // true console.log(false == 0); // true // boolean vs string → string → number, then boolean → number console.log(true == "1"); // true ("1" → 1, true → 1)

JavaScript internally uses Number() conversion for strings and booleans when using ==. This can lead to surprising results if not understood properly.

📌 తెలుగు : == ఆపరేటర్ type coercion (implicit conversion) చేస్తుంది — compare చేయడానికి ముందు values ని ఒక common type (సాధారణంగా Number) లోకి మారుస్తుంది. ఉదా: 5 == "5" → true (ఎందుకంటే "5" ని Number(5) గా మార్చి compare చేస్తుంది). true == 1 → true (true ని 1 గా మారుస్తుంది).

🎯 4. Strict Equality (===) — No Coercion, Type Matters

The === operator compares both value AND type. If types are different, it returns false immediately — no type conversion happens.

console.log(5 === "5"); // false (number vs string) console.log(true === 1); // false (boolean vs number) console.log(null === undefined); // false (different types) console.log(0 === false); // false (number vs boolean)
✅ Best Practice: Always prefer === over == to avoid unexpected type coercion bugs. Use == only when you intentionally want type coercion.

📌 తెలుగు : === ఆపరేటర్ value మరియు type రెండింటినీ compare చేస్తుంది. Types different అయితే వెంటనే false return చేస్తుంది — ఎలాంటి type conversion జరగదు. Best practice: ఎప్పుడూ === ని వాడండి, unexpected bugs ని avoid చేయడానికి.

⚠️ 5. Special Comparisons: null, undefined, NaN, "", false

These are common tricky cases that appear in technical interviews and MCQ rounds.

null == undefined → true
null === undefined → false
"" == 0 → true ("" → 0)
"" === 0 → false
false == "" → true (both falsy)
false === "" → false
null == false → false
null === false → false
undefined == 0 → false
NaN == NaN → false (always!)
"true" == true → false ( "true" → NaN, true → 1 )
"01" == 1 → true
console.log(null == undefined); // true console.log(null === undefined); // false console.log("" == 0); // true (empty string → 0) console.log(NaN == NaN); // false (NaN is not equal to anything, even itself) console.log(false == ""); // true (both falsy, after coercion) console.log(null == false); // false
🔥 Key rules: null and undefined are equal only with == (not ===). NaN is never equal to NaN. Empty string converts to 0. Falsy values (false, 0, "") can be equal under coercion.

📌 తెలుగు : ప్రత్యేక సందర్భాలు:
• null == undefined → true, కానీ null === undefined → false.
• NaN == NaN → false (ఎప్పటికీ).
• Empty string ("") ని == తో 0 తో compare చేస్తే true వస్తుంది (ఎందుకంటే "" → 0).
• false == "" → true (రెండూ falsy values, coercion తర్వాత). ఈ cases ఇంటర్వ్యూలలో చాలా ముఖ్యమైనవి.

📖 6. Summary: == vs === and Best Practices

Quick Reference:

// == : allows type coercion (converts types) // === : does NOT allow coercion (types must match) console.log(5 == "5"); // true (coerced) console.log(5 === "5"); // false (types differ) console.log(true == 1); // true console.log(true === 1); // false console.log(null == undefined); // true console.log(null === undefined); // false console.log("" == 0); // true console.log("" === 0); // false
✅ Kalyan's Advice: Use === by default. It makes your code predictable and avoids hidden coercion bugs. Only use == when you explicitly want type conversion (e.g., checking if a value is null or undefined: value == null).

Remember: Every comparison returns a Boolean (true/false). Mastering these operators will save you hours of debugging.

📌 తెలుగు సారాంశం :
• == → type coercion చేస్తుంది (values ని convert చేసి compare చేస్తుంది).
• === → type coercion చేయదు (types మ్యాచ్ అవ్వాలి).
• ఎప్పుడూ === ని వాడండి, అది safe మరియు predictable. == ని specific cases లో మాత్రమే వాడండి (ఉదా: value == null).
