📌 1. Declaration vs Initialization

In JavaScript, creating a variable can be split into two steps:

  • Declaration: telling JavaScript that a variable exists (let a; or var a;)
  • Initialization: assigning a value to the variable (a = 100;)
// Declaration and initialization in one line let a = 100; // Same thing in two lines let b; // Declaration b = 200; // Initialization
💡 Understanding this split is essential to grasp hoisting.

📌 తెలుగు : JavaScript లో variable creation ని రెండు steps గా split చేయవచ్చు:
• Declaration: variable ఉందని JavaScript కి తెలియజేయడం (let a;)
• Initialization: variable కి value assign చేయడం (a = 100;)
Hoisting ని అర్థం చేసుకోవడానికి ఈ split చాలా ముఖ్యం.

⚠️ 2. var vs let: Accessing Before Declaration
// Using var - NO error, prints undefined console.log(x); // undefined var x = 100; console.log(x); // 100 // Using let - ERROR! Cannot access before initialization console.log(y); // ReferenceError let y = 200;

Why does var print undefined but let throws an error? The answer is hoisting.

📌 తెలుగు : var తో declaration కి ముందు access చేస్తే undefined వస్తుంది. let తో అలా చేస్తే ReferenceError వస్తుంది. ఎందుకు? Hoisting వలన.

🧠 3. Hoisting Explained: Creation Phase & Execution Phase

When a JavaScript program runs, the Global Execution Context (GEC) is created on the stack. The GEC has two phases:

  • Creation Phase (Memory Allocation): JavaScript scans for declarations, moves them to the top (hoisting), and allocates memory.
  • Execution Phase: Code runs line by line (assignments, function calls, console.log).
// Original code console.log(a); var a = 100; console.log(a); // What happens during Creation Phase (hoisting) var a; // Declaration hoisted to top // Execution Phase starts console.log(a); // a is undefined (var gets default value) a = 100; // initialization console.log(a); // 100
🔥 Key Point: var variables are hoisted AND automatically initialized with undefined. let variables are hoisted but NOT initialized.

📌 తెలుగు : JavaScript program start అయినప్పుడు Global Execution Context (GEC) stack లో create అవుతుంది. GEC కి రెండు phases:
• Creation Phase: Declarations ని top కి move చేస్తుంది (hoisting), memory allocate చేస్తుంది.
• Execution Phase: Code line by line run అవుతుంది.
var hoisted అయ్యాక undefined తో initialize అవుతుంది. let hoisted అవుతుంది కానీ initialize కాదు.

⏳ 4. Temporal Dead Zone (TDZ) with let

The Temporal Dead Zone is the region from the start of the block until the variable is initialized. In this zone, the variable exists (hoisted) but cannot be accessed.

// TDZ starts here (from line 1) console.log(x); // ReferenceError - in TDZ let x = 100; // TDZ ends here (initialization) console.log(x); // Works fine (100)
💡 TDZ Diagram:
[Declaration] → [TDZ starts] → ... (cannot access) ... → [Initialization] → [TDZ ends] → can access.

Only let and const have TDZ. var does NOT have a TDZ.

📌 తెలుగు : Temporal Dead Zone (TDZ) అంటే block ప్రారంభం నుండి variable initialized అయ్యే వరకు ఉన్న region. ఈ region లో variable ఉంది (hoisted), కానీ access చేయలేము. let మరియు const కి TDZ ఉంటుంది. var కి TDZ లేదు.

📊 5. var vs let: Key Differences

var

  • ✅ Hoisted
  • ✅ Initialized with undefined automatically
  • ✅ No Temporal Dead Zone
  • ❌ Can be accessed before declaration (gets undefined)
  • ⚠️ Function-scoped

let

  • ✅ Hoisted
  • ❌ NOT initialized (remains empty/temporal dead zone)
  • ✅ Has Temporal Dead Zone
  • ❌ Cannot access before initialization (ReferenceError)
  • ✅ Block-scoped
// var example console.log(usingVar); // undefined (no error) var usingVar = 10; // let example console.log(usingLet); // ReferenceError: Cannot access before initialization let usingLet = 20;
🧠 Kalyan's Advice: Always prefer let over var to avoid unexpected behavior. var can lead to subtle bugs due to hoisting and lack of TDZ.

📌 తెలుగు : var vs let: var hoisted అవుతుంది, undefined తో initialize అవుతుంది, TDZ లేదు. let hoisted అవుతుంది కానీ initialize కాదు, TDZ ఉంటుంది. ఎప్పుడూ let ని prefer చేయండి — unexpected bugs ని avoid చేయడానికి.

📞 6. Function Hoisting (Function Declarations)

Function declarations are fully hoisted — you can call them before they are defined.

// Calling before declaration - WORKS! greet(); // Output: "Hello, Kalyan!" function greet() { console.log("Hello, Kalyan!"); }

How it works: During creation phase, the entire function declaration is hoisted to the top. So the function is available throughout the scope.

🔥 Note: Function expressions (assigning function to variable) are NOT hoisted like declarations. Only function declarations are fully hoisted.
// This will NOT work greetExpr(); // TypeError: greetExpr is not a function var greetExpr = function() { console.log("Expression"); };

📌 తెలుగు : Function Declarations పూర్తిగా hoisted అవుతాయి — define చేయడానికి ముందే call చేయవచ్చు. Function Expressions (variable కి assign చేసిన function) hoisted కావు. అందుకే declarations ని వాడటం safe.

📖 7. Summary: Hoisting Essentials
  • Hoisting = moving declarations to the top of their scope during the creation phase.
  • var: Hoisted + initialized with undefined → accessible before declaration (but value is undefined).
  • let & const: Hoisted but NOT initialized → Temporal Dead Zone (TDZ) → accessing before initialization throws ReferenceError.
  • Function Declarations: Fully hoisted (can call before definition).
  • Best Practice: Declare variables at the top of their scope. Use let and const over var.
// Good practice let name = "Kalyan"; console.log(name); // Instead of console.log(age); // undefined (confusing) var age = 25;
🎯 Understanding hoisting and TDZ will save you hours of debugging and make you a confident JavaScript developer.

📌 తెలుగు సారాంశం :
• Hoisting: Declarations ని top కి move చేయడం (creation phase లో).
• var: Hoisted + undefined తో initialized → declaration కి ముందు access చేయవచ్చు (undefined వస్తుంది).
• let/const: Hoisted but NOT initialized → TDZ ఉంటుంది → declaration కి ముందు access చేస్తే ReferenceError.
• Function Declarations: Fully hoisted.
• Best Practice: Variables ని scope ప్రారంభంలో declare చేయండి. var కంటే let/const ని వాడండి.
