📌 1. Recap: The Synchronous Blocking Problem

JavaScript is single-threaded and synchronous by default. A long-running task blocks the entire call stack, freezing the UI.

function fetchCourseProgress() { // 5 second delay (busy-wait) — BLOCKS everything! } fetchCourseProgress(); // UI freezes for 5 seconds fetchLeaderboard(); // waits for previous to complete
🔥 Single-threaded = one statement at a time. Blocking = slow tasks stop everything else.

📌 తెలుగు : JavaScript డిఫాల్ట్ గా single-threaded మరియు synchronous. Long-running task call stack ని block చేస్తుంది → UI freeze అవుతుంది.

🔄 2. Solution: Web APIs + Callback Queue + Event Loop
🧠 JavaScript Engine (Call Stack + Heap)
+
🌐 Web APIs (provided by Browser/Node.js): setTimeout, DOM APIs, fetch, etc.
+
📦 Callback Queue (FIFO)
+
⚡ Event Loop (monitors stack & queue)
✅ Web APIs allow us to offload blocking tasks. When they complete, callbacks go to the Callback Queue. The Event Loop moves them to the Call Stack when it's empty.

📌 తెలుగు : Web APIs (setTimeout, fetch, DOM APIs) blocking tasks ని offload చేస్తాయి. Task complete అయిన తర్వాత callback Callback Queue లోకి వెళ్తుంది. Event Loop call stack empty ఉన్నప్పుడు queue నుండి stack కి తరలిస్తుంది.

⏰ 3. setTimeout: Execute Later (Non-blocking)
console.log("Start"); setTimeout(() => { console.log("Inside setTimeout (after 2 sec)"); }, 2000); console.log("End"); // Output: // Start // End // (after 2 seconds) Inside setTimeout
💡 setTimeout does NOT block! The callback is registered with Web API, timer runs outside the call stack. Meanwhile, the rest of the code executes.

📌 తెలుగు : setTimeout non-blocking. Callback Web API కి register అవుతుంది, timer stack వెలుపల నడుస్తుంది. మిగిలిన code immediate గా execute అవుతుంది. Timer complete అయిన తర్వాత callback queue → event loop → call stack.

🎯 4. Event Loop in Action: Multiple setTimeout (Different Timers)
setTimeout(() => console.log("Task A (5 sec)"), 5000); setTimeout(() => console.log("Task B (2 sec)"), 2000); console.log("Synchronous code"); // Output: // Synchronous code // (after 2 sec) Task B (2 sec) // (after 5 sec) Task A (5 sec)
🔥 Order depends on timer, not code position. Whichever timer finishes first gets added to the callback queue first → executes first (when stack is empty).

📌 తెలుగు : Multiple setTimeout లలో, ఎవరి timer ముందు complete అవుతుందో వారి callback queue లో ముందు వస్తుంది → ముందు execute అవుతుంది. Timer duration (milliseconds) execution order ని నిర్ణయిస్తుంది.

🔄 5. setInterval: Repeated Execution (with clearInterval)
let count = 0; let intervalId = setInterval(() => { count++; console.log(`Fetching data... (${count})`); }, 2000); // Stop after 10 seconds setTimeout(() => { clearInterval(intervalId); console.log("Stopped fetching."); }, 10000);
✅ setInterval executes callback repeatedly every delay milliseconds.
✅ clearInterval(id) stops it.
✅ Use setTimeout to schedule the stop after a duration.

📌 తెలుగు : setInterval ప్రతీ delay milliseconds తర్వాత callback ని repeat చేస్తుంది. clearInterval(id) ద్వారా stop చేయవచ్చు. setTimeout ద్వారా కొంత సమయం తర్వాత clearInterval call చేయడం common pattern.

⏱️ 6. setTimeout(fn, 0) - Defer to End of Stack
console.log("A"); setTimeout(() => console.log("B (setTimeout 0)"), 0); console.log("C"); // Output: A, C, B
💡 Even with 0ms delay, the callback is placed in the callback queue and executes after the current synchronous code finishes.

📌 తెలుగు : setTimeout(fn, 0) callback ని callback queue లో వేస్తుంది. Current synchronous code complete అయిన తర్వాతే execute అవుతుంది.

📖 7. Summary: Async JavaScript Essentials
  • JavaScript is synchronous & single-threaded by nature (Call Stack only).
  • Web APIs (setTimeout, setInterval, fetch, DOM events) provide async capabilities.
  • Callbacks from Web APIs go to Callback Queue (FIFO).
  • Event Loop continuously checks: if Call Stack empty → pushes callback from queue to stack.
  • setTimeout schedules execution after delay (non-blocking).
  • setInterval repeats execution; clearInterval stops it.
  • Zero-delay setTimeout defers execution until after synchronous code.
// Mental model // Call Stack ← Event Loop ← Callback Queue ← Web APIs (timers, etc.)
🧠 Kalyan's Advice: Practice tracing async code manually. Predict output order. Understand that setTimeout delay is MINIMUM time, not exact execution time (depends on stack).

📌 తెలుగు సారాంశం :
• JavaScript synchronous + single-threaded (Call Stack only).
• Web APIs (setTimeout, fetch) async capabilities ఇస్తాయి.
• Callbacks → Callback Queue (FIFO).
• Event Loop: Stack empty అయితే queue నుండి stack కి callback ని తరలిస్తుంది.
• setTimeout → non-blocking delay.
• setInterval → repeat; clearInterval → stop.
• Zero-delay setTimeout → synchronous code తర్వాత execute అవుతుంది.
Async code ని mental గా trace చేయడం ప్రాక్టీస్ చేయండి.
