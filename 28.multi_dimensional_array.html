📊 Multi-Dimensional Arrays in JavaScript

📦 1D, 2D, 3D Arrays | Nested Arrays | Memory (References) | Nested Loops for Traversal — by Kalyan (Telugu WebTech)
📌 1. What Does "Dimension" Mean in Arrays?

Dimension = number of pieces of information required to access an element.

  • 1D Array: Need 1 piece of info (index) → e.g., list of student names.
  • 2D Array: Need 2 pieces of info (row, column) → e.g., days × time (weather data).
  • 3D Array: Need 3 pieces of info (center, floor, classroom) → e.g., students per classroom.
💡 Don't confuse with geometry! Dimension here means "how many indices do you need to specify?"

📌 తెలుగు : Dimension = element ని access చేయడానికి అవసరమైన information ముక్కల సంఖ్య.
1D → ఒక్క index (student names list). 2D → రెండు indices (rows, columns) — weather data (day × time). 3D → మూడు indices (center, floor, classroom). ఇది geometry కాదు, "ఎన్ని indices కావాలి" అనేదే dimension.

🔹 1. One-Dimensional Array (1D)
let students = ["Joe", "Jack", "Jim", "Jill", "Jiffy"]; // Access: students[index] console.log(students[0]); // Joe console.log(students[2]); // Jim
✅ Access pattern: arr[index] — one index needed.

📌 తెలుగు : 1D array: ఒకే index తో access. ఉదా: students[0], students[2].

🔹 2. Two-Dimensional Array (2D) - Weather Data
// 3 days, each day has 3 readings (morning, afternoon, evening) let weather = [ [25, 30, 22], // Day 0: morning, afternoon, evening [23, 31, 24], // Day 1 [26, 30, 20] // Day 2 ]; // Access: weather[dayIndex][timeIndex] console.log(weather[0][0]); // 25 (Day0, morning) console.log(weather[1][2]); // 24 (Day1, evening)
🔥 Memory: Outer array stores references (addresses) to inner arrays. Each inner array is a separate object in Heap.
// Nested loop to iterate 2D array for (let i = 0; i < weather.length; i++) { for (let j = 0; j < weather[i].length; j++) { console.log(`Day ${i}, Time ${j}: ${weather[i][j]}`); } }

📌 తెలుగు : 2D array = array of arrays. Weather example: 3 days × 3 times. Access: weather[day][time]. Memory లో outer array inner arrays యొక్క references (addresses) ని store చేస్తుంది. Nested loops (i, j) తో traverse చేస్తాం.

🔹 3. Three-Dimensional Array (3D) - Students per Classroom
// 2 centers, each has 2 floors, each floor has 2 classrooms let students = [ [ // Center 0 [50, 70], // Floor 0: classroom0=50, classroom1=70 [65, 87] // Floor 1: classroom0=65, classroom1=87 ], [ // Center 1 [90, 45], // Floor 0 [80, 30] // Floor 1 ] ]; // Access: students[center][floor][classroom] console.log(students[0][0][0]); // 50 console.log(students[1][1][1]); // 30
✅ 3D access pattern: arr[i][j][k] — three indices.
// Nested loops for 3D array for (let i = 0; i < students.length; i++) { // centers for (let j = 0; j < students[i].length; j++) { // floors for (let k = 0; k < students[i][j].length; k++) { // classrooms console.log(`Center ${i}, Floor ${j}, Classroom ${k}: ${students[i][j][k]} students`); } } }

📌 తెలుగు : 3D array: centers × floors × classrooms. Access: students[center][floor][classroom]. Nested loops (i, j, k) తో traverse చేస్తాం — ఎన్ని dimensions ఉంటే అన్ని loops.

🧠 4. Critical Memory Concept: Arrays Store References

When you create a nested array (array inside array), the outer array does NOT contain the inner array directly. Instead, it stores the reference (memory address) of the inner array.

let outer = [[10, 20], [30, 40]]; // outer[0] stores ADDRESS of [10,20] (not the array itself) // outer[1] stores ADDRESS of [30,40]
🔥 Why this matters: This reference-based model is crucial for understanding shallow copy vs deep copy. Copying outer array copies references, not inner arrays.

📌 తెలుగు : Nested array లో, outer array inner array ని నేరుగా store చేయదు — దాని reference (memory address) ని store చేస్తుంది. ఇది shallow copy vs deep copy ని అర్థం చేసుకోవడానికి చాలా ముఖ్యం. Copying outer array అంటే references ని copy చేయడం, inner arrays ని కాదు.

📐 5. Traversing Irregular Multi-Dimensional Arrays

Inner arrays can have different lengths. Always use array[i].length for dynamic traversal.

let jagged = [ [1, 2], // length 2 [3, 4, 5, 6], // length 4 [7, 8, 9] // length 3 ]; for (let i = 0; i < jagged.length; i++) { for (let j = 0; j < jagged[i].length; j++) { console.log(jagged[i][j]); } } // Outputs all elements: 1,2,3,4,5,6,7,8,9
💡 Best practice: Always use arr[i].length inside the inner loop — it automatically adapts to different inner array sizes.

📌 తెలుగు : Inner arrays different lengths ఉండొచ్చు. Dynamic గా traverse చేయడానికి ఎప్పుడూ arr[i].length వాడండి. ఇది inner array size ని అనుసరిస్తుంది.

📖 6. Quick Reference: Multi-Dimensional Arrays
  • 1D: [10, 20, 30] → access: arr[i]
  • 2D: [[10,20],[30,40]] → access: arr[i][j]
  • 3D: [[[1,2],[3,4]],[[5,6],[7,8]]] → access: arr[i][j][k]
  • Nested loops: Number of loops = number of dimensions.
  • Memory: Outer stores references (addresses) to inner arrays.
  • Dynamic lengths: Use arr[i].length for inner loop conditions.
// Creating 2D array dynamically let rows = 3, cols = 4; let matrix = []; for (let i = 0; i < rows; i++) { matrix[i] = []; for (let j = 0; j < cols; j++) { matrix[i][j] = i * cols + j + 1; } } console.log(matrix);
🧠 Kalyan's Advice: Practice creating and traversing 2D and 3D arrays. Draw memory diagrams on paper. This skill is essential for data structures like matrices, grids, and real-world applications.

📌 తెలుగు సారాంశం :
• 1D: arr[i]
• 2D: arr[i][j] (nested loops)
• 3D: arr[i][j][k] (three loops)
• Memory: Outer stores references to inner arrays.
• Dynamic lengths: arr[i].length వాడండి.
