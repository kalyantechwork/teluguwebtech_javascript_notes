📌 1. Introduction: Using a Free API (Cat Facts)
🐱 Free API: https://catfact.ninja/fact → returns a random cat fact in JSON format.

Example response from browser:

{ "fact": "Cats sleep 16 hours a day", "length": 26 }
🔥 The browser can display JSON directly, but we want to fetch this data inside our JavaScript application. That's where the fetch() function comes in.

📌 తెలుగు : Free API ఉదాహరణ: https://catfact.ninja/fact → random cat fact ని JSON లో return చేస్తుంది. Browser నేరుగా JSON ని చూపిస్తుంది, కానీ మనం ఈ data ని మన JavaScript application లోకి తీసుకురావాలి. దీనికి fetch() function ఉపయోగిస్తాం.

📡 2. Basic fetch() with Promises (.then, .catch)

fetch() returns a Promise. We use .then() for success and .catch() for network errors.

fetch('https://catfact.ninja/fact') .then(response => { // response is a ReadableStream (raw bytes) return response.json(); // also returns a Promise }) .then(data => { console.log(data); // JavaScript object console.log(data.fact); }) .catch(error => { console.log('Fetch failed:', error); });
✅ Key points:
  • fetch() returns a Promise → resolve = response object.
  • response.json() also returns a Promise → parses JSON body.
  • Network errors (no internet, wrong domain) go to .catch().

📌 తెలుగు : fetch() ఒక Promise ని return చేస్తుంది. .then() success కి, .catch() network errors కి. response.json() కూడా Promise ని return చేస్తుంది — ఇది JSON body ని parse చేస్తుంది.

📨 3. Understanding the Response Object
fetch('https://catfact.ninja/fact') .then(response => { console.log(response.status); // 200 (OK) console.log(response.ok); // true console.log(response.headers); // Headers object return response.json(); }) .then(data => console.log(data));
💡 The first .then() gives a Response object (headers, status, etc.). The actual JSON body must be read using .json() or .text().

📌 తెలుగు : Response object లో status (200), ok (true), headers వంటివి ఉంటాయి. JSON body ని చదవడానికి .json() లేదా .text() వాడాలి.

✨ 4. fetch() with async/await (Cleaner Syntax)
async function getCatFact() { try { const response = await fetch('https://catfact.ninja/fact'); const data = await response.json(); // wait for JSON parsing console.log(data.fact); console.log(data.length); } catch (error) { console.log('Error:', error); } } getCatFact();
🧠 Advantages:
  • No nested .then() → linear, synchronous-looking code.
  • try/catch handles both fetch and .json() errors.
  • Much easier to read and maintain.

📌 తెలుగు : async/await వలన code linear గా, synchronous లాగా కనిపిస్తుంది. try/catch fetch మరియు .json() errors రెండింటినీ handle చేస్తుంది. చదవడానికి చాలా సులభం.

⚠️ 5. Handling HTTP Errors (404, 500, etc.)

Important: fetch() only rejects on network failures. HTTP errors (404, 500) still resolve the Promise. You must check response.ok.

async function fetchWithErrorCheck() { try { const response = await fetch('https://catfact.ninja/factt'); // invalid endpoint if (!response.ok) { throw new Error(`HTTP error! Status: ${response.status}`); } const data = await response.json(); console.log(data); } catch (error) { console.log('Caught:', error.message); } } fetchWithErrorCheck();
🔥 Best practice: Always check response.ok (true for 200-299). Throw an error for non-OK statuses to trigger your catch block.

📌 తెలుగు : fetch() network failures మీద మాత్రమే reject చేస్తుంది. HTTP errors (404, 500) వచ్చినా Promise resolve అవుతుంది. response.ok ని check చేసి, non-OK status కి error throw చేయాలి.

📖 6. Summary: fetch() Methods Comparison
// Promise chain version fetch(url) .then(res => res.json()) .then(data => console.log(data)) .catch(err => console.log(err)); // Async/await version (recommended) async function fetchData() { try { const res = await fetch(url); if (!res.ok) throw new Error(`Status: ${res.status}`); const data = await res.json(); console.log(data); } catch (err) { console.log(err); } }
📌 Remember:
  • fetch() always returns a Promise.
  • Use .json() or .text() to read the body (both return Promises).
  • Check response.ok for HTTP errors.
  • async/await + try/catch is cleaner and more readable.

📌 తెలుగు సారాంశం : • fetch() ఎప్పుడూ Promise ని return చేస్తుంది.
• Body ని చదవడానికి .json() లేదా .text() (ఇవి కూడా Promises).
• HTTP errors కోసం response.ok check చేయండి.
• async/await + try/catch → cleaner, more readable code.
