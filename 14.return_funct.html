📚 Functions in JavaScript | 4 Types with Memory Tracing

🔧 Global Execution Context (GEC) | Stack (EC for functions) | Parameters, Return, Function Call — by Kalyan (Telugu WebTech)
Type 1 No Input · No Output (No parameters, No return)

This function performs a task but does not accept any arguments and does not return any value.

let a = 100; let b = 200; function add() { let c = a + b; // uses outer variables console.log(c); // prints but doesn't return } add(); // function call — output: 300

Memory trace (Stack):

📌 Global Execution Context (GEC) created → allocates memory for a, b
📌 a=100, b=200
📌 add() called → new Execution Context (EC) for add pushed on stack
📌 Inside add: c = 100+200 = 300 → console.log(300) prints 300
📌 add finishes → EC popped, c destroyed
📌 GEC finishes → program ends, stack empty
✅ Without a function call, the function body never executes. Calling is mandatory.

📌 తెలుగు : ఈ ఫంక్షన్ ఎలాంటి input తీసుకోదు, ఎలాంటి output return చేయదు. కేవలం task perform చేస్తుంది. Memory trace: Global Execution Context (GEC) లో a, b allocate అవుతాయి. add() call అయినప్పుడు add కి కొత్త Execution Context stack లో push అవుతుంది. c = 300, console.log prints 300. add అయిపోయాక EC pop అవుతుంది. ఫంక్షన్ call చేయకపోతే execute అవ్వదు.

Type 2 No Input · Returns Output

Function does not take parameters but computes and returns a value using return keyword.

let a = 100, b = 200; function add() { let c = a + b; return c; // returns value to caller } let result = add(); // result collects returned value console.log(result); // 300

Memory trace:

📌 GEC: a=100, b=200, result initially undefined
📌 add() called → new EC for add on stack
📌 Inside add: c = 300 → return c (value 300 sent back)
📌 add EC popped (c destroyed)
📌 Returned value 300 assigned to result in GEC
📌 console.log(result) → prints 300
🔥 Important: return immediately exits the function and sends the value back. No code after return executes.

📌 తెలుగు : ఫంక్షన్ parameters తీసుకోదు, కానీ return statement ద్వారా value ని caller కి పంపుతుంది. Memory trace: GEC లో a, b, result. add() EC లో c=300, return c (value 300 returns). add EC pop అవుతుంది. result variable కి 300 assign అవుతుంది. return వెంటనే function ని exit చేస్తుంది.

Type 3 Accepts Parameters · No Return

Function takes parameters (inputs) but does not return anything; typically performs an action like printing.

function add(x, y) { // x and y are parameters let c = x + y; console.log(c); } let a = 100, b = 200; add(a, b); // arguments: values of a and b are passed // Output: 300

Memory trace (Stack execution):

📌 GEC: a=100, b=200
📌 add(a,b) called → new EC for add pushed
📌 Parameters x, y allocated: x = 100, y = 200
📌 Inside add: c = 300 → console.log(300)
📌 add finishes → EC popped (x, y, c destroyed)
📌 Back to GEC → no further code → GEC popped
✅ Parameters are local variables that receive values from arguments during function call.

📌 తెలుగు : ఫంక్షన్ parameters తీసుకుంటుంది, కానీ return చేయదు. Memory trace: add(x,y) call అయినప్పుడు x, y parameters కి a,b విలువలు assign అవుతాయి (x=100, y=200). c = 300, console.log prints. add EC pop అయిన తర్వాత x, y, c delete అవుతాయి. Parameters local variables లాగా పనిచేస్తాయి.

Type 4 Accepts Parameters · Returns Output

Most common and powerful type: takes input, processes, and returns a value without side effects (pure function).

function add(x, y) { let c = x + y; return c; // returns the sum } let a = 100, b = 200; let result = add(a, b); console.log(result); // 300

Memory trace (self-practice assignment):

📌 GEC: a=100, b=200, result undefined
📌 add(a,b) called → EC for add created, x=100, y=200
📌 c = 300 → return c (value 300 returned)
📌 add EC popped, c, x, y destroyed
📌 Returned 300 assigned to result in GEC
📌 console.log(result) → 300 printed
📌 GEC popped
🧠 Assignment for you: Trace the above code with stack diagrams on paper. Understand how return transfers value and how EC is destroyed. This builds strong fundamentals.

📌 తెలుగు : ఇది అత్యంత ప్రాముఖ్యమైన type: input తీసుకుంటుంది, process చేసి return చేస్తుంది. Memory trace: GEC లో a,b,result. add EC లో x=100, y=200, c=300, return c (value 300 returns). add EC pop అవుతుంది. result కి 300 assign. console.log prints 300. ఈ tracing ని మీరే paper మీద చేసి practice చేయండి — ఇది fundamentals ని బాగా strengthen చేస్తుంది.

🧠 5. Deep Dive: Global Execution Context (GEC) & Call Stack

Whenever a JavaScript program starts, the Global Execution Context (GEC) is created and pushed onto the call stack. It has two phases:

  • Creation Phase: Memory allocated for variables (hoisting), functions are stored.
  • Execution Phase: Code runs line by line, assignments happen, function calls create new Execution Contexts.
let a = 10; let b = 20; function multiply(x, y) { return x * y; } let result = multiply(a, b); console.log(result);

Call Stack behavior:

1. GEC pushed → a, b, result, multiply allocated (creation phase)
2. Execution: a=10, b=20, then multiply(a,b) called
3. New EC for multiply pushed → x=10, y=20 → return 200
4. multiply EC popped → returned value assigned to result
5. console.log(200) → GEC pops → stack empty
💡 The call stack keeps track of which function is currently executing. Top of stack = currently running function.

📌 తెలుగు : JavaScript program start అయినప్పుడు Global Execution Context (GEC) call stack లో push అవుతుంది. ఇందులో రెండు phases: Creation Phase (memory allocation) మరియు Execution Phase (code execution). Function call అయినప్పుడు ఆ function కి కొత్త Execution Context stack లో push అవుతుంది. Function అయిపోయాక pop అవుతుంది. Stack లో top లో ఉన్నదే ప్రస్తుతం execute అవుతున్న code.

⚡ 6. Important: Function Definition vs Function Call
// DEFINITION (code is NOT executed yet) function greet() { console.log("Hello!"); } // CALL (executes the function body) greet(); // Output: Hello! // Without call, nothing happens
🔥 Remember: Defining a function is like writing a recipe. Calling it is like actually cooking. Multiple calls = cook the same recipe multiple times.

Functions can be called any number of times. Each call creates a new independent Execution Context.

📌 తెలుగు : Definition: function ని రాయడం (code execute అవ్వదు). Call: function ని functionName() తో execute చేయడం. ప్రతీ call కి కొత్త independent Execution Context stack లో create అవుతుంది. ఒక function ని ఎన్నిసార్లు అయినా call చేయవచ్చు.
